# ステージエディタ

ブラウザ上で動作する3Dレベルエディタ。配置物・敵・ボス・プレイヤー開始位置を配置し、
`stageData.json` として書き出す。

仕様の詳細は [stage_editor.md](../../docs/design/stage_editor.md) を参照。

---

## 起動方法

⚠️ **`index.html` をダブルクリックしても動かない。** ES Modules と importmap を使っているため、
`file://` では CORS で読み込みがブロックされる。**同梱のローカルサーバー経由で開くこと。**

```sh
# リポジトリのルートで実行（既定ポート 8091）
python tools/serve_editor.py

# ブラウザで開く
# http://127.0.0.1:8091/tools/stage-editor/
```

このサーバーは静的配信に加えて、エディタの「保存 (上書き)」からの書き込みを受け付け、
`assets/data/stage-test.json` を**直接上書き**する（ダウンロードして手動配置する必要がない）。

> 素の `python -m http.server` でも閲覧・編集はできるが、その場合「保存」は上書きできず
> **ダウンロードにフォールバック**する。上書き保存を使うには `serve_editor.py` で起動すること。

> three.js は CDN（jsdelivr）から読み込んでいるため、**初回起動時はインターネット接続が必要**。
> オフラインで使いたい場合は `three.module.js` と `examples/jsm/controls/TransformControls.js` を
> `vendor/` へ落として importmap のパスを差し替える。

## 操作方法

### カメラ（Unityのシーンビュー準拠）

| 操作 | 動作 |
|---|---|
| 右ドラッグ | 視点回転 |
| 右ドラッグ中の `WASD` | 前後左右へ移動 |
| 右ドラッグ中の `Q` / `E` | 下降 / 上昇 |
| 右ドラッグ中の `Shift` | 移動速度アップ |
| 中ドラッグ | 平行移動（パン） |
| ホイール | ズーム |
| `F` | 選択中オブジェクトにフォーカス |

### オブジェクト

| 操作 | 動作 |
|---|---|
| 左クリック | 選択（何もない場所で選択解除） |
| `W` / `E` / `R` | 移動 / 回転 / 拡縮モード |
| `Ctrl` を押しながらドラッグ | スナップ（移動100 / 回転15° / 拡縮100） |
| `Ctrl + D` | 複製 |
| `Delete` | 削除 |
| `Ctrl + S` | 保存（`stage-test.json` を上書き。ブラウザの「ページを保存」は無効化される） |
| `Ctrl + Z` / `Ctrl + Y` | Undo / Redo |
| Global / Local ボタン | ギズモの軸を切り替え（**坂の編集では Local 推奨**） |
| Hierarchy をダブルクリック | 選択してフォーカス |

### 制約

- **光源（Light）は位置と Range・Color のみ**。回転・拡縮は効かない（範囲はInspectorの `Range` で編集する）
  - ワイヤーフレームの球が光の届く範囲。パレットの「光 …」から色プリセット付きで追加できる
- **敵・ボス・プレイヤー開始位置は拡縮できない**（サイズは `enemies/*.json` 側の責務）
- **敵・ボス・プレイヤー開始位置の回転は Y のみ**
- **ボスとプレイヤー開始位置は削除・複製できない**（各1つ）
- 坂は配置物の Rotation X / Z を傾けて作る

## 入出力

| ボタン | 動作 |
|---|---|
| stageData 読込 | 既存の `assets/data/stageData.json` を読み込んで編集を再開 |
| カタログ読込 | `assets/data/stageCatalog.json` を手動で読み込んでパレットを差し替え（下記の通り通常は自動） |

> **カタログは起動時に自動読込される。** ローカルサーバー経由なら `assets/data/stageCatalog.json` を
> `fetch` して常に最新へ同期する。取得に失敗した場合（`file://` 起動やファイル欠損）のみ、
> HTML内の埋め込みカタログにフォールバックする。手動の「カタログ読込」は差し替え検証用。
| 保存 (JSON書き出し) | `stageData.json` をダウンロード。**手動で `assets/data/` へ配置する** |

**起動時は編集対象の `assets/data/stage-test.json` を自動で読み込む。** 保存先と同じファイルなので、
開き直しても実ファイルの内容がそのまま出る（エディタの表示とファイルが食い違わない）。

`localStorage` への自動保存は、ファイルを取得できなかった場合のフォールバックとして使う。
未保存の変更はリロードで失われるため、区切りごとに「保存 (上書き)」すること。

## 座標系

JSONに書き出される座標は **DxLib の座標系（左手系・Y-up）**。
three.js は右手系のため、シーン内で `scale.z = -1` のコンテナを噛ませて吸収している
（詳細は [stage_editor.md](../../docs/design/stage_editor.md) の「3-2. 座標系」）。

ビューポート左下のステータス表示はDxLib座標で出している。

> **未検証**：エディタで置いた座標がゲーム内で本当に同じ位置に出るかは、
> ゲーム側の `props[]` 読み込みを実装した時点で必ず確認すること。
> 特に**回転（坂）の符号**は実機で確かめる必要がある。
