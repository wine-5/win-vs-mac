# C++ 変数・メンバ変数の宣言順序

## クラスのメンバ変数はサイズの大きい順に並べる

メモリのパディング（無駄な隙間）を減らし、メモリ効率を上げるため。

---

## 推奨する宣言順序

### 1. ポインタ型（x64環境では8バイト）
```cpp
int* ptr;
Player* pPlayer;
```

### 2. 8バイト型
```cpp
double d;
```

### 3. 4バイト型
```cpp
int a;
float b;
```

### 4. 2バイト型
```cpp
short s;
```

### 5. 1バイト型
```cpp
char c;
bool flag;
```

---

## 具体例

```cpp
// ❌ 悪い例（12バイト）
class Bad
{
    char  a;   // 1バイト + 3パディング
    int   b;   // 4バイト
    char  c;   // 1バイト + 3パディング
};

// ✅ 良い例（8バイト）
class Good
{
    int   b;   // 4バイト
    char  a;   // 1バイト
    char  c;   // 1バイト + 2パディング
};
```

---

## なぜパディングが発生するのか

CPUはメモリを「自分のサイズの倍数のアドレス」から読む仕組みになっている。

```
int（4バイト）  → 4の倍数のアドレスに配置される
char（1バイト） → どこでもOK
double（8バイト）→ 8の倍数のアドレスに配置される
```

パディングは変数ごとに追加される（合計してから追加されるわけではない）。

---

## 優先的に見直すべきクラス

大量に生成されるクラスほど効果が大きい。

| 優先度 | 対象 | 理由 |
|--------|------|------|
| 高 | 敵、弾、パーティクルなど | 1000個生成されると差が1000倍に |
| 中 | 毎フレーム処理されるもの | 処理頻度が高い |
| 低 | 1個しか存在しないもの（Player、GameManagerなど） | 数バイトの差は誤差レベル |

---

## sizeof で確認する方法

Visual Studio のウォッチウィンドウで確認できる。

```
sizeof(クラス名のインスタンス)
```

デバッグ中にウォッチウィンドウへ入力するだけで即確認可能。

---

## ポインタのサイズに注意

ポインタのサイズは環境によって異なる。

| 環境 | ポインタサイズ |
|------|--------------|
| x64（64bit） | 8バイト |
| x86（32bit） | 4バイト |

現在の開発環境（x64）ではポインタは8バイトなので、doubleと同じ扱いで先頭に置く。
