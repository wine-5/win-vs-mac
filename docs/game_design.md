# ゲームデザイン詳細

## ゲームの流れ
1. タイトル画面
2. ファイル選択画面（エクスプローラーからファイル・フォルダを選択）
3. ロード画面（ファイル解析・ダンジョン生成・プロセス列挙・インストール済みアプリ検索）
4. ゲーム画面
5. リザルト画面（失敗） or クリア画面（ボス討伐）
6. タイトルに戻る

---

## シーン一覧
| シーン名 | 概要 |
|---|---|
| タイトル | ゲーム開始画面 |
| ファイル選択 | エクスプローラー風UIでフォルダを選択・難易度をリアルタイム表示 |
| ロード | ファイル解析・ダンジョン生成・プロセス列挙・インストール済みアプリ検索 |
| ゲーム | メインのダンジョン攻略画面 |
| ポーズ | ゲーム中の一時停止画面 |
| リザルト（失敗） | ゲームオーバー画面 |
| クリア | ボス討伐後の画面 |

## ファイル選択画面
Windowsのエクスプローラーをゲーム内に再現したレトロUI。
選択したフォルダをリアルタイムで解析して推定難易度を表示する。
```
┌─────────────────────────────────┐
│ DUNGEON DRIVE - SELECT FOLDER   │
├─────────────────────────────────┤
│ 📁 デスクトップ                  │
│ 📁 ドキュメント                  │
│ 📁 ダウンロード                  │
│ 📁 ピクチャ                      │
│ 💾 C:/                          │
│ 💾 D:/                          │
├─────────────────────────────────┤
│ 選択中: C:/Users/○○/Desktop     │
│ ファイル数: 24   階層: 3        │
│ 推定難易度: ██░░░░ EASY         │
└─────────────────────────────────┘
```

## 戦闘システム
参考：ハデス（アイソメトリックアクション）

| 項目 | 内容 |
|---|---|
| 視点 | アイソメトリック（斜め上からの3D俯瞰） |
| 移動 | WASDで自由移動 |
| 通常攻撃 | 選択したファイルの種類で攻撃方法が変わる |
| スキル | ゲーム中にエクスプローラーを呼び出してファイルを選択して発動 |
| ダッシュ | Shiftキーで短距離ダッシュ |
| 部屋制 | フォルダ構造で生成された部屋をクリアして進む |
| 敵 | 複数の敵がリアルタイムで動き回る |
| ボス | `mac_os.exe`（Macをモチーフにしたオリジナルボス） |

## ファイルと武器・パラメータの対応（検討中）

| ファイル種類 | ゲームへの反映 |
|---|---|
| `.exe` | 攻撃力が高い近距離武器 |
| `.mp4 / .png` | 防御系アイテム |
| `.txt` | 素早さ上昇・連続攻撃 |
| ファイルサイズ大 | 重量級・高威力 |
| フォルダの階層の深さ | ダンジョンの階層数に反映 |

## 敵の設計

### 敵の生成方針
レジストリを参照してインストール済みのアプリを検索し敵として出現させる。
ロード画面中に処理を行い「PCを解析中...」「エディタを検出中...」などの
進捗メッセージを表示する。

### 敵の分類

| プロセス・アプリ | 敵のキャラクター |
|---|---|
| `chrome.exe`（ブラウザ系） | 重くて鈍い大型の敵 |
| `csrss.exe`（システム系） | 素早い小型の敵 |
| Defender（セキュリティ系） | 防御力が高い敵 |
| `devenv.exe`（Visual Studio） | 重くて起動が遅い大型の敵 |
| `Code.exe`（VSCode） | 軽くて素早い・拡張機能数で強化される |
| `rider64.exe`（Rider） | バランス型の強敵 |
| `cursor.exe`（Cursor） | AI系の予測不能な動きをする敵 |

### 最終ボス
Macをモチーフにしたオリジナルボス。
名前は`mac_os.exe`などWindows風のファイル名で表現。
（Appleの商標・ロゴは使用しない）

## ダンジョン生成

### 生成ルール
フォルダ構造を解析してダンジョンを自動生成する。

| フォルダの情報 | ダンジョンへの反映 |
|---|---|
| フォルダ名 | 部屋の名前 |
| フォルダ内のファイル数 | 部屋の広さ |
| サブフォルダの数 | 分岐する通路の数 |
| フォルダの階層の深さ | ダンジョンの深さ（階層数） |

### 正規化ルール
環境依存を防ぐため生のデータを難易度帯に変換する。

**階層の深さ**
| 実際の階層数 | ダンジョンの階層数 | 難易度 |
|---|---|---|
| 1〜3階層 | 1階層 | Easy |
| 4〜7階層 | 3階層 | Normal |
| 8階層以上 | 5階層（上限） | Hard |

**部屋の数**
- ファイル数・フォルダ数が多くても最大20部屋に固定

## Windows API連携

| 取得情報 | ゲームへの反映 |
|---|---|
| プロセス列挙 | 敵の種類・数の生成 |
| CPU使用率 | 敵の強さ・行動速度のリアルタイム変化 |
| メモリ使用量 | ダンジョンの環境変化 |
| ファイルシステム | マップ・武器・パラメータの生成 |
| レジストリ | インストール済みアプリの検索 |

## ゲーム要素（検討中）

### 心拍数システム
- プレイヤーの心拍数という概念をゲーム内に追加
- CPU使用率またはネットワーク送受信量をリアルタイムで取得して反映
- 心拍数が高いほどゲームに何らかの影響を与える（詳細検討中）
