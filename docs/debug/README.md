# デバッグ用コード ドキュメント

このフォルダは、**リリース時に削除するデバッグ専用コード**を機能ごとに記録するための場所です。

## 目的

デバッグ用に追加したコードは本番ビルドに含めたくないため、最終的にすべて削除します。
ただし削除箇所が複数ファイルに散らばると消し忘れ・消しすぎが起きやすいので、
機能を追加するたびにこのフォルダへ「どこに・何を足したか／どう消すか」を md で残します。

## 運用ルール

- デバッグ機能を1つ追加したら、`<機能名>.md` を新規作成する（例: `debug_camera.md`）。
- コード側のデバッグ箇所には必ず `DEBUG:` で始まるコメントを付ける（grep で洗い出せるようにするため）。
- md には最低限、以下を書く。
  - 機能の概要と操作方法
  - 追加したコミットID
  - 削除・復元すべきファイルと箇所の一覧
- リリース前に、このフォルダの各 md を見ながら削除作業を行う。

## 一括検索の目安

削除対象はコード全体を次のように検索すると洗い出せる。

```bash
git grep -n "DEBUG:"
```
