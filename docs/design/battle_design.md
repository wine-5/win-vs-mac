# 戦闘システム設計書

## 確定事項

---

### 戦闘形式

リアルタイムアクション（ハデス・ゼルダ的）

---

### 基本アクション

| アクション | 操作 |
|---|---|
| 移動 | WASD |
| 通常攻撃 | 選択中の武器で攻撃 |
| 武器切り替え | 1・2・3キー |
| ダッシュ | Shiftキー |

---

### 武器システム（ロードアウト方式）

ゲーム開始前にエクスプローラーからファイルを3つ選択してスロットに割り当てる。
戦闘中は1・2・3キーで武器を切り替えて戦う。
同じ拡張子のファイルを複数選択することも可能。
その場合はファイルサイズなどの情報をもとにAIが自動的に差別化する。

```
[ゲーム開始前 - ファイル選択画面]
スロット1: document.txt  → 素早い連続攻撃
スロット2: game.exe      → 重い一撃
スロット3: photo.png     → 防御スキル

[戦闘中]
1キー → txtの武器に切り替え
2キー → exeの武器に切り替え
3キー → pngの武器に切り替え
```

---

### 武器生成（AI API方式）

ファイル選択時にAI APIを使って武器パラメータを自動生成する。
使用APIはClaude AIまたはGemini API（未確定）。
APIキーは暗号化して管理する（詳細は未定）。

#### 生成タイミング（ハイブリッド方式）

1. **ファイル選択時（即時・ローカル処理）**
   - ファイル名・拡張子からローカルで簡易プレビューを表示
   - 例：`.exe` → 「攻撃系」、`.png` → 「防御系」程度の表示
   - 拡張子ごとの属性判定ルールはローカルで定義

2. **確定後（API生成・ロード画面）**
   - ロード画面でAPIを呼び出して詳細パラメータを生成
   - 「武器を生成中...」などの演出を表示

#### APIプロンプト形式（例）

```
以下のファイル情報から武器を1つ生成してください。
必ずJSON形式のみで返してください。

ファイル名: project_final_v3.exe
ファイルサイズ: 2.4MB

出力形式:
{
  "weapon_name": "武器名",
  "attack_type": "slash / pierce / magic / defense のいずれか",
  "attack_power": 1〜100の整数,
  "speed": 1〜100の整数,
  "special_effect": "特殊効果の説明（1文）"
}
```

#### 生成例

```json
{
  "weapon_name": "終焉の断片剣",
  "attack_type": "slash",
  "attack_power": 78,
  "speed": 45,
  "special_effect": "finalの名を冠し、3回に1度クリティカルが発動する"
}
```

### パラメータ設計（2層構造）

武器を切り替えてもプレイヤーの基礎パラメータは変化しない。
武器のボーナスが基礎パラメータに加算される形。

```
最終的なダメージ = プレイヤーの基礎攻撃力 + 装備中の武器の攻撃力ボーナス
```

#### プレイヤー基礎パラメータ（レベルアップで上昇）

| パラメータ | 説明 |
|---|---|
| HP | プレイヤーの体力 |
| 攻撃力 | 敵に与えるダメージの基礎値 |
| 防御力 | 敵から受けるダメージの軽減値 |
| 移動速度 | 移動の速さ |

レベルシステムあり。敵を倒すごとに経験値を獲得しレベルが上がる。
レベルアップ時の具体的な数値上昇量はレベルデザイン時に調整する。

#### 武器パラメータ（AI APIが生成）

| パラメータ | 説明 |
|---|---|
| 武器名 | AIがファイル名から命名 |
| 攻撃タイプ | slash / pierce / magic / defense |
| 攻撃力ボーナス | 基礎攻撃力に加算される値（1〜100） |
| 攻撃速度 | 攻撃の速さ（1〜100） |
| 特殊効果 | 固有の能力（AIが1文で定義） |

### ダンジョン生成

#### 方式
ルールベース（AI APIは使用しない）。
フォルダの情報を解析してダンジョンのパラメータに変換する。

#### ステージ構成（5ステージ固定）

| ステージ | フォルダ | 難易度イメージ |
|---|---|---|
| 1 | デスクトップ | Easy |
| 2 | ドキュメント | Normal |
| 3 | ダウンロード | Normal〜Hard |
| 4 | ピクチャ | Hard |
| 5 | Cドライブ | Very Hard |

#### 難易度の決定

各フォルダの中身を解析して難易度を自動決定する。
プレイヤーのPC環境によって同じステージでも難易度が変わる。
「自分のPCの状態がそのまま難易度になる」体験を重視する。

| フォルダの情報 | ゲームへの反映 |
|---|---|
| ファイル数 | 敵の数 |
| フォルダの階層の深さ | ダンジョンの部屋数 |
| 合計サイズ | ボス敵の強さ |
| ファイルの種類の多様性 | 敵の種類の多様性 |

敵の数・強さの上限と下限はバトルシステム完成後のレベルデザイン時に調整する。
